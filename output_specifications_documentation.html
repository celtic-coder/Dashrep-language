Directives versus actions versus phrases

Phrase

A phrase is a hyphenated phrase that has a definition, and the definition is inserted in place of the hyphenated phrase.  Phrases are recognized regardless of whether or not they appear within parameter brackets.

Directive

A directive is a hyphenated phrase that triggers a special functionality in the translation.  Directives are not recognized if they appear within parameter brackets.

Action

An action is a hyphenated phrase that has special significance when it appears at the beginning of text that is enclosed by parameter brackets.  An action is not recognized if it is not within parameter brackets, or if it is not the first hyphenated phrase in the parameter brackets.

Hyphens, spaces, lines, & square brackets

hyphen-here

A directive that inserts a hyphen, with no adjacent spaces.  Internally this phrase is defined as no-space - no-space.

character-hyphen

A phrase that contains one hyphen character (-).

four-hyphens

A phrase that contains four adjacent hyphen characters (----).

no-space

A directive that removes any spaces (concatenation) after all other phrases have been replaced.  If the one-space directive is adjacent, the no-space directive is ignored.

empty-text

A pre-defined phrase that contains no text.  It is useful with the action copy-from-phrase-to-phrase-and-replace-string-in-phrase-with-phrase.  To clear a phrase, use the action clear-phrase.  To test whether a phrase's definition is empty, use the action yes-or-no-empty-phrase.

one-space

A directive that inserts a single space.  If the no-space directive is adjacent, the no-space directive is ignored.  Repeating this directive does not insert multiple adjacent spaces, just a single space.  If multiple spaces are needed, a phrase such as four-spaces can be defined as multiple spaces, and then those spaces will be inserted where the string appears.

character-space

A phrase that contains one space character.  This phrase is useful when using the action copy-from-phrase-to-phrase-and-replace-string-in-phrase-with-phrase.

non-breaking-space

A directive that inserts a single space — as defined in a target language.  This string is inserted even if the no-space directive is adjacent.  The default, which is intended for use in HTML, is &nbsp;.

new-line

A directive that inserts a newline character, which specifies the start of a new line.  If the target language, such as HTML, ignores line breaks, the line break will not be visible in the displayed result.

empty-line

A directive that specifies an empty line, which is equivalent to two adjacent new-line directives.

line-break

A phrase that should be defined as a text string that, in turn, specifies a line break in the target language.  For example, if the target language is HTML, normally this phrase would be defined as <br />.

character-newline

A phrase that contains a newline character.  In some other programming languages this character is represented by the string \n.

character-open-square-bracket

A phrase that contains the open-square-bracket character ([).

character-close-square-bracket

A phrase that contains the close-square-bracket character (]).

character-tab

A phrase that contains a tab character.  In some other programming languages this character is represented by the string \t.

delayed-nospace

This phrase is not defined, but this name is reserved for use as a placeholder where the phrase no-space needs to be inserted later, after other replacements have been done.  When ready, this phrase should be defined as no-space just before the final translation step.

delayed-newline

This phrase is not defined, but this name is reserved for use as a placeholder where the phrase new-line needs to be inserted later, after other replacements have been done.  When ready, this phrase should be defined as new-line just before the final translation step.

Definition-related phrases

list-of-phrases-newly-defined

A phrase that contains a list of all the hyphenated phrases that have been imported or defined.  This definition can be cleared to limit the list to phrases that were imported or defined after the definition was cleared.

dashrep-definitions-begin

A directive that specifies that the text that follows are phrases and definitions.  This phrase cannot be nested.  This change continues until the phrase dashrep-definitions-end is encountered.

dashrep-definitions-end

A directive that specifies the end of phrases and definitions that began with the directive dashrep-definitions-begin.

dashrep-definition-append-next

This directive requests that the next definition be appended to the existing definition of the specified phrase.  This directive allows a list to be created during the reading of definitions because it overrides the normal convention of replacing the existing definition (when the same phrase name is defined more than once).

define-end

A directive that functions the same as several hyphens (----) at the end of a definition.  It is provided as a speakable alternative for ending a definition.

define-begin

This directive is ignored, but is defined for symmetry so that it can be used after the directive define-end.  It is not needed because the context already determines where a definition begins.

clear-phrase

An action that clears the definition for the specified phrase.  This action is equivalent to using the equal-sign notation without any text for the new definition.

clear-phrases-listed-in-phrase

An action that clears the definitions for the phrases that are named in the specified phrase.

clear-all-dashrep-phrases

An action that clears all the definitions of all the phrases.  Remember that this action resets every definition of every phrase, except the phrases that are defined internally.

delete-phrases-listed-in-phrase

An action that deletes (removes) the phrases that are named in the specified phrase.  This phrase is related to the action clear-phrases-listed-in-phrase except that instead of just clearing the definition, this delete version removes (deletes) the phrase so that it will not be recognized.  This action is useful for creating a template in which the listed phrases will be replaced by varying definitions at later times.

copy-from-phrase-to-phrase

An action that copies the definition of the first hyphenated phrase into the definition of the second hyphenated phrase.  This action must be followed by two phrase names, not one.  The previous definition of the second hyphenated phrase is deleted.  This action is useful in situations where the text includes hyphens, line breaks, multiple adjacent spaces and any other text that cannot be assigned using the equal-sign approach.

copy-to-phrase-from-following-text

An action that copies into the first hyphenated phrase the text that follows the first hyphenated phrase, up to the end of the parameter brackets.  The text can contain hyphens if they are not adjacent to an open angle bracket or close angle bracket.  Trailing spaces and trailing hyphens are ignored.  This action is useful in situations where a phrase name is needed, but that phrase name must not be expanded yet.

append-from-phrase-to-phrase

An action that appends the definition of the first phrase to the end of the definition of the second phrase, and inserts a single space between the existing text and the appended text.  The two phrase names can be the same.

append-from-phrase-to-phrase-no-space

An action that appends the definition of the first phrase to the end of the definition of the second phrase, without inserting a separating space.  The two phrase names can be the same.

prepend-from-phrase-to-phrase

An action that prepends the definition of the first phrase into the beginning of the definition of the second phrase, without inserting a separating space.

prepend-from-phrase-to-phrase-no-space

An action that prepends the definition of the first phrase into the beginning of the definition of the second phrase, and inserts a single space between the prepended text and the existing text.

append-new-line-to-phrase

An action that appends a newline code to the end of a definition.

insert-definition-of-phrase

An action that inserts the definition of the specified phrase.  This action provides a way to immediately expand a phrase by just one level of recursion, without also expanding parameter phrases within the definition.

expand-phrase-to-phrase

An action that expands ("translates") the contents of the first phrase and puts the results into the second phrase.  This action provides a way to translate many different chunks of text rather than translating all text using the same translation choices.  It is especially useful if the text contains hyphens but is not Dashrep code.

expand-only-parameters-in-phrase-to-phrase

An action that expands ("translates") any parameters in the first phrase and puts the results into the second phrase.  Unlike the related action expand-phrase-to-phrase, this action does not expand any phrases (except the starting named phrase) that appear outside of parameter brackets.  This action is especially useful if there needs to be a delay in expanding the non-parameter phrases.

copy-multiple-words-in-phrase-to-phrases-named-in-pattern

An action that copies multiple phrase definitions.  Specifically space-delimited words in the first phrase are copied to multiple phrase names, where the phrase names have the pattern specified in the second phrase.  The pattern must contain characters between the string ambee- (or conambee-) and the string -amenn (or amenncon-), and these characters must be the name of a phrase that contains the space-delimited words that will be inserted at that position within the pattern.  If the phrase that contains the target word list contains only a string without any spaces (a single word), then only one item will be copied.  If that list is empty, nothing will be copied.  If the number of words in the first phrase is different from the number of words in the list that is specified within the pattern, the copying will stop when the shorter list is exhausted.  As an example of this action, if the first phrase contains dolphin buffalo bear, and the secomd phrase contains item-ambee-list-of-numbers-as-text-amenn, and the phrase list-of-numbers-as-text contains the string one two three, then the phrase item-one will contain dolphin and the phrase item-two will contain buffalo and the phrase item-three will contain bear.  This action is useful for copying a long list fast, without needing to use a separate Dashrep action for each item copied.

append-multiple-from-phrases-named-in-pattern-to-phrase

An action that appends multiple phrase definitions onto the end of the definition of the second phrase.  The pattern for the multiple phrase names is specified in the first phrase.  The pattern must contain characters between the string ambee- (or conambee-) and the string -amenn (or amenncon-), and these characters must be the name of a phrase that contains the space-delimited words that will be inserted at that position within the pattern.  If the phrase that contains the word list contains only a string without any spaces (a single word), then only one item will be appended.  If the list is empty, nothing will be appended.  As an example of this action, if the first phrase contains the string item-ambee-list-of-numbers-as-text-amenn and the phrase list-of-numbers-as-text contains the string one two three and the second phrase already contains the string dolphin and the phrase item-one contains buffalo and the phrase item-two contains bear and the phrase item-three contains eagle then the second phrase will contain dolphin buffalo bear eagle.  This action is useful for building a long list fast, without needing to use a separate Dashrep action for each item appended.

dashrep-yes-append-not-replace-for-imported-phrases

If this phrase equals yes, and if a phrase definition is being imported from a file, and if that phrase has the same name as an existing phrase, then the definition is appended to the end of the existing definition, rather than replacing the existing definition.  A space is inserted to separate the two definitions.  This capability is useful when multiple files contain phrase definitions with the same names, and those definitions are lists that should be combined.

dashrep-yes-or-no-export-delimited-definitions

If this phrase equals yes then the action write-all-dashrep-definitions-to-file or the action write-all-dashrep-phrase-names-to-file or the action write-dashrep-definitions-listed-in-phrase-to-file action is used, this phrase name (enclosed in brackets) is inserted at the end of each phrase name.  Otherwise exported definitions are written using the normal Dashrep convention.  This capability enables the exported definitions to be converted into special formats such as HTML or XML.

export-defs-all-begin

If the phrase dashrep-yes-or-no-export-delimited-definitions equals yes when the action write-all-dashrep-definitions-to-file or the action write-all-dashrep-phrase-names-to-file or the action write-dashrep-definitions-listed-in-phrase-to-file is used, this phrase name (enclosed in brackets) is inserted at the beginning of all the written-to-a-file phrase definitions.  This capability is useful if the written definitions need to be translated into a different format.

export-defs-all-end

If the phrase dashrep-yes-or-no-export-delimited-definitions equals yes when the action write-all-dashrep-definitions-to-file or the action write-all-dashrep-phrase-names-to-file or the action write-dashrep-definitions-listed-in-phrase-to-file action is used, this phrase name (enclosed in brackets) is inserted at the end of all the written-to-a-file phrase definitions.  This capability is useful if the written definitions need to be translated into a different format.

export-defs-def-begin

If the phrase dashrep-yes-or-no-export-delimited-definitions equals yes when the action write-all-dashrep-definitions-to-file or the action write-all-dashrep-phrase-names-to-file or the action write-dashrep-definitions-listed-in-phrase-to-file action is used, this phrase name (enclosed in brackets) is inserted at the beginning of each definition.  (A separate phrase is inserted at the end of the phrase name).  This capability is useful if the written definitions need to be translated into a different format.

export-defs-def-end

If the phrase dashrep-yes-or-no-export-delimited-definitions equals yes when the write-all-dashrep-definitions-to-file or write-all-dashrep-phrase-names-to-file or write-dashrep-definitions-listed-in-phrase-to-file action is used, this phrase name (enclosed in brackets) is inserted at the end of each definition.  (A separate phrase is inserted at the beginning of the phrase name).  This capability is useful if the written definitions need to be translated into a different format.

export-defs-phrase-begin

If the phrase dashrep-yes-or-no-export-delimited-definitions equals yes when the write-all-dashrep-definitions-to-file or write-all-dashrep-phrase-names-to-file or write-dashrep-definitions-listed-in-phrase-to-file action is used, this phrase name (enclosed in brackets) is inserted at the beginning of each phrase name.  (A separate phrase is inserted at the end of the definition).  This capability is useful if the written definitions need to be translated into a different format.

export-defs-phrase-end

If the phrase dashrep-yes-or-no-export-delimited-definitions equals yes when the write-all-dashrep-definitions-to-file or write-all-dashrep-phrase-names-to-file or write-dashrep-definitions-listed-in-phrase-to-file action is used, this phrase name (enclosed in brackets) is inserted at the end of each phrase name.  (A separate phrase is inserted at the beginning of the definition).  This capability is useful if the written definitions need to be translated into a different format.

Decision-related phrases

if-yes-begin

An action that looks at the first space-delimited word, and if it is yes (or contains that character string) then this action removes all the text between the next occurrences of [-if-else-] and [-if-end-] (and removes the [-if-else-] and [-if-end-] text strings).  Otherwise (if the first word that follows does not contain yes) then this action removes all the text between itself (the action name) and the next occurrence of [-if-else-] (and removes the [-if-else-] and [-if-end-] text strings.  This action provides a major branching capability.  However, the if-yes-begin action and the [-if-else-] text string and the [-if-end-] text string cannot be nested (as allowed in most programming languages).  If nested branching is needed, each level of branching must be done in a different phrase definition (which improves documentation if meaningful phrase names are used).

if-no-begin

An action that works similar to the action if-yes-begin except that the string no is used, instead of the string yes.

if-else

A special text string that is recognized by the if-yes-begin action and the if-no-begin action.  It must be enclosed by parameter brackets in order to be recognized.

if-end

A special text string that is recognized by the if-yes-begin action and the if-no-begin action.  It must be enclosed by parameter brackets in order to be recognized.

yes-or-no-empty-phrase

An action that supplies yes if the following specified phrase is empty, and supplies no otherwise.  A phrase is empty if it contains nothing, or if it contains only spaces, or if it has not yet been defined.

yes-or-no-same-two-phrase-definitions

An action that supplies yes if the following two phrase contain the same text, and supplies no otherwise.

yes-or-no-same-two-words

An action that supplies yes if the following two space-delimited words are the same, and supplies no otherwise. If either word might be empty, then a warning is put into the trace information because the number of operands is not two.

yes-or-no-word-is-in-phrase

An action that supplies yes if the following first word is one of the words in the following phrase, and supplies no otherwise. If the first word is a phrase name in parameter brackets and that phrase is empty, then a warning is put into the trace information because the number of operands is not two.

yes-or-no-opposite

An action that supplies no if the text that follows contains the text string yes, and otherwise supplies yes.

yes-if-any-yes

An action that is the same as the action yes-if-not-no.

no-if-any-no

An action that is the same as the action no-if-not-yes.

yes-if-not-no

An action that supplies no if the following word (space-delimited text string) is no, and supplies yes otherwise.  This action is useful if the word can be something other than yes or no, but only no is significant.

no-if-not-yes

An action that supplies yes if the following word (space-delimited text string) is yes, and supplies no otherwise.  This action is useful if the word can be something other than yes or no, but only yes is significant.

Number-related phrases

zero-or-nonzero

An action that supplies either zero or nonzero depending upon whether the following integer number equals zero.  This phrase is useful when a value of zero requires special handling.

zero-one-multiple

An action that supplies either zero or one or multiple depending upon whether the following integer number equals zero, one, or multiple (more than one).  This phrase is very useful for choosing between singular and plural forms of words based on a data-supplied count.

numeric-equal-greater-less-compare

An action that supplies the word equal if the two numbers that follow are numerically equal, or supplies the word greater if the first number is greater than the second number, or supplies the word less if the first number is less than the second number.

numeric-add

An action that add the numbers that follow, and supplies the result.  The numbers can be positive or negative integers, or they can be positive or negative decimal numbers, but floating-point numbers are not recognized.  As applies to all Dashrep numeric calculations, each calculation converts the text-based digits into binary values, and the result is converted back into text-based digits, so the calculations are slow compared to the speed of most programming languages.  If more than a few simple calculations are being done, consider doing the intensive numerical processing in a different programming language.

numeric-minus

An action that subtracts the second number from the first number, and supplies the result.  See numeric-add for additional information.

numeric-multiply

An action that multiplies the first number times the second number, and supplies the result.  See numeric-add for additional information.

numeric-divide-by

An action that divides the first number by the second number, and supplies the result.  See numeric-add for additional information.

numeric-maximum

An action that determines the maximum value of the numbers that follow, and supplies the result.  See numeric-add for additional information.

numeric-minimum

An action that determines the minimum value of the numbers that follow, and supplies the result.  See numeric-add for additional information.

numeric-increment

An action that increments the integer value in the named phrase.  This action does not supply the result.  This number is incremented each time this special action is encountered.  Use of this action should be avoided.  Usually the action use-template-and-parameters-to-create-simple-list-with-name (or the full list version) and the createlist-item-number phrase, or the action unique-value can be used instead.

numeric-decrement

Decrements (reduces by one) the integer value in the named phrase.  This number is decremented each time this special action is encountered.  Use of this action should be avoided.  It is useful for debugging situations that loop many more times than expected.

numeric-integer

An action that calculates the integer value of a real number, and supplies the result.  If an integer is supplied, the same integer is returned (although leading zeros, if present, are removed).

numeric-absolute

An action that calculates the absolute value of the number that follows, and supplies the result.

numeric-odd-or-even

An action that supplies the word odd or even based on whether the value of the following operand is an odd or even integer.  It is useful for handling row (or column) numbers in a table.  The result is unpredictable if the operand is not a positive integer.

numeric-sine

An action that calculates the (trigonometric) sine of the number that follows, and supplies the result.

numeric-cosine

An action that calculates the (trigonometric) cosine of the number that follows, and supplies the result.

numeric-logarithm-base-e

An action that calculates the base-e logarithm of the specified number, and supplies the result.

numeric-logarithm-base-10

An action that calculates the base-10 logarithm of the specified number, and supplies the result.

numeric-pi

An action that supplies the text representation for the value of pi.

numeric-vector-add-number

An action that adds a single number to each number in the supplied vector and produces a vector sum.  Each vector is a space-delimited list of numbers.  The vector in the definition of the first operand is added to the single number that is supplied as the second operand.  The resulting vector is put into the definition of the third operand.  The numbers can be positive or negative integers, or they can be positive or negative decimal numbers, but floating-point numbers are not recognized.  If subtraction is needed, use the action numeric-vector-multiply-by-number to multiply one of the vectors by minus one, and then add the resulting vector to the positive-sign vector.  As applies to all Dashrep numeric calculations, each calculation converts the text-based digits into binary values, and the result is converted back into text-based digits, so the calculations are slow compared to the speed of most programming languages.

numeric-vector-multiply-by-number

An action that multiplies a vector times a single number and produces a resulting vector.  Each vector is a space-delimited list of numbers.  The vector in the definition of the first operand is multiplied by the second operand, and the resulting vector is put into the definition of the third operand.  The numbers can be positive or negative integers, or they can be positive or negative decimal numbers, but floating-point numbers are not recognized.  As applies to all Dashrep numeric calculations, each calculation converts the text-based digits into binary values, and the result is converted back into text-based digits, so the calculations are slow compared to the speed of most programming languages.

numeric-vectors-add

An action that adds two vectors and produces a vector sum.  Each vector is a space-delimited list of numbers.  The vector in the definition of the first operand is added to the vector in the definition of the second operand, and the resulting vector is put into the definition of the third operand.  The numbers can be positive or negative integers, or they can be positive or negative decimal numbers, but floating-point numbers are not recognized.  As applies to all Dashrep numeric calculations, each calculation converts the text-based digits into binary values, and the result is converted back into text-based digits, so the calculations are slow compared to the speed of most programming languages.

numeric-vector-integers

An action that converts into integers the real numbers that are supplied in the vector that is named in the first operand.  The vector is a space-delimited list of numbers, which can include integers as well as real numbers.  The resulting vector is put into the definition of the second operand.  Floating-point numbers are not recognized.

numeric-vector-absolutes

An action that converts a list of numbers within a vector into their absolute values, and puts the resulting list into the definition of the second operand.  The vector is a space-delimited list of numbers, which can include integers as well as real numbers.  Floating-point numbers are not recognized.

numeric-vectors-from-delta-values-calculate-distances

An action that gets the list of space-delimited delta X integer numbers in the definition of the first operand and gets the list of space-delimited delta Y integer numbers in the definition of the second operand, and calculates a list of integer distances that are put into the definition of the third operand.  The numbers can be positive or negative integers.  Although decimal numbers can be supplied, the resulting values are truncated to integer values, so the decimal portion is lost.  Floating-point numbers are not recognized.  As applies to all Dashrep numeric calculations, each calculation converts the text-based digits into binary values, and the result is converted back into text-based digits, so the calculations are slow compared to the speed of most programming languages.

numeric-two-dimensional-sort-into-columns-and-rows

XML-import phrases

xml-move-attributes-into-tag-elements-within-phrase

An action that moves the content of an attribute tag or a specify tag inside the preceding tag.  For example, it converts the string <img /><attribute href="image.jpg"><attribute alt="image"> into the string <img href="image.jpg" alt="image" />.  The word specify can be used instead of attribute.  This functionality is useful for generating XML and HTML code by using hyphenated phrases such as image-here specify-href image.jpg specify-end specify-alt image specify-end (instead of having to specify the nesting of the beginning and ending of a tag, such as image-here-begin and image-here-end).

linewise-translate-xml-tags-in-file-to-dashrep-phrases-in-file

Translates XML in the specified input file into Dashrep phrases and puts the results in the specified output file.  In the resulting text, phrase names are inserted where XML tags appeared.  XML tags are converted into phrase names, where the phrase name is a concatenation (joined version) of the XML tag names.  For example, begin-xml-peoplelist-person-name and end-xml-peoplelist-person-name surrounds text that was tagged with the XML tag structure <xml><peoplelist><person><name>.  The prefix begin-and-end- is used for self-terminating XML tags such as <br />.  If the resulting phrase is defined, the definition is inserted instead.  If the non-tag content contains any hyphens, they are replaced with the phrase hyphen-here.  If a tag's opening bracket (<) and closing bracket (>) are not both on the same line, the tag will not be recognized.  The resulting file, combined with appropriate definitions for those phrase names, enables an XML or HTML file to be converted into nearly any kind of text file.  The location of the target file is specified by the phrase dashrep-path-prefix-for-file-writing.

dashrep-first-xml-tag-name

A phrase that supplies an XML tag name (without angle brackets) that indicates the first XML tag to be recognized when using the action i-b linewise-translate-xml-tags-in-file-to-dashrep-phrases-in-file i-e.  Tags at higher levels within the XML structure are ignored.  This tag name becomes the first tag name in the generated phrase names.

dashrep-list-of-xml-phrases

A phrase that is filled with a list of all the phrase names that were written by the action i-b linewise-translate-xml-tags-in-file-to-dashrep-phrases-in-file i-e.

dashrep-xml-level-reset-if-zero

If this user-defined phrase equals 0 (zero) then the XML handling process resets the translation process so that another XML file can be processed without being affected by effects that arose during the previous translation.

dashrep-xml-trace-on-yes-or-no

If this user-defined phrase equals yes, then messages are written to the phrase dashrep-debug-trace-log during an XML translation, and if this phrase equals no (or any other value), those messages are not written.

dashrep-xml-yes-ignore-if-no-tag-replacement

If this user-defined phrase equals yes then XML translations ignore data that would be tagged with a hyphenated phrase that is not defined.  For example, if the context for XML data is <xml><dolphin><firstname>Flipper</firstname></dolphin></xml> and the phrase xml-dolphin-firstname has not been defined, then this XML data is ignored.  The match is case sensitive, which means that a defined phrase of XML-Dolphin-Firstname would not be sufficient to capture the data in this example.

Time-related phrases

get-current-time-in-epoch-seconds

Gets the current time as an integer number that indicates the number of seconds since January 1, 1970 at midnight GMT (UTC).  This convention matches the convention used in Linux and Windows, but not the convention used by Apple.

split-epoch-seconds-into-named-components

Uses the supplied epoch seconds to put appropriate numbers into the phrases that begin with the word time.  The results are specific to local time at the server location.  Simple Dashrep definitions can use these values to easily generate nearly any date and time format.  See also split-epoch-seconds-into-named-components-for-zero-meridian, which applies to the time at zero meridian.

split-epoch-seconds-into-named-components-for-zero-meridian

Uses the supplied epoch seconds to put appropriate numbers into the phrases that begin with the word time.  The results are specific for UTC.  Addition and substraction can be used to convert to a specific time zone.  Simple Dashrep definitions can use these values to easily generate nearly any date and time format.  See also split-epoch-seconds-into-named-components, which applies to local time at the server.

time-day-of-month

time-day-of-week

time-minute

The minute of the hour, based on the epoch seconds supplied to the action split-epoch-seconds-into-named-components.

time-month-number

Advanced phrases

dashrep-undefined

A phrase that contains the text that is inserted in some cases when the result of an action is undefined.  The default value is dashrep_undefined.

dashrep-endless-loop-counter-limit

A user-defined phrase that specifies the maximum number of replacements allowed during a Dashrep translation.  This number can be redefined such as to a larger number.  If this limit is exceeded, Dashrep translation is stopped and an error message indicates which phrase was replaced the most number of times.  If large amounts of data are processed, the default number is not going to be large enough.  This limit interrupts endless loops, which can occur if a phrase is defined in a way that (either directly or indirectly) includes its own name.

dashrep-endless-loop-count

An action that supplies the current value of the internal endless-loop counter.  This special action, which has no operands, supplies the count on a read-only basis.

insert-codeview-tags-into-phrase

Inserts into the specified phrase definition special tags that enable the raw Dashrep code to be displayed, such as on a web page.  The special inserted tags, such as <dashrep_codeview_action_begin> and <dashrep_codeview_action_middle_1>, are expected to be replaced by the strings in phrases such as dashrep-codeview-action-begin and dashrep-codeview-action-middle-1.

end-of-line-here

A special phrase that contains a unique value each time this phrase appears in parameter brackets.  Currently this action simply supplies a number that is incremented, but future versions may use non-sequential numbers (to accomodate parallel processing).

dashrep-list-of-recognized-phrase-names

An action that gets at all the defined phrases, and puts those phrase names into the second phrase, with a space between each phrase name.  For security purposes this action can be disabled.

dashrep-yes-indicate-line-endings

If this phrase is defined as yes, the actions copy-from-file-to-phrase and copy-from-file-to-phrases-line-numbered insert the text end-of-line-here at line-break positions.  The end-of-line-here phrase does not have a default definition.

yes-or-no-use-faster-subset-expand

If this phrase is defined as yes, a newer and faster section of code is used to expand Dashrep code (that does not contain parameter brackets).  This code handles the directives no-space and one-space and new-line and empty-line.  This code is useful for increasing the speed of a server response for a website.  The original code is still available for uses that require handling all Dashrep conventions and that do not require fast execution times.

dashrep-yes-do-not-expand-special-phrases
